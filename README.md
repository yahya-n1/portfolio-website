PART 2: DESIGN

1. What design style do you want to represent you best? (minimalist, creative, professional, etc)
In addition to a very distinctive minimalistic and professional design of my website, I want to encourage interaction with the content on my site while making it extremely easy for visitors to find and read the content on my site.

2. What will your final color schemes be and what significance will they have on your overall design?
The color scheme used for my site is dark blue and white; I have used this color combination because dark blue connotes professionality, reliability, and high-tech, and white will create a contrast with the dark blue and help to keep the layout cleanly arranged such that it is easy to read.

3. What fonts do you plan on using for the headings & the body text?
The website uses a neutral, clean sans serif typeface like Arial for all its heading, body, and other text. Sans serif fonts are popularly used for modern website designs because they provide good readability when read on computers or devices without support for serif fonts.

4. What does your design reflect about you and/or your profession? 
The design demonstrates my area of study (Computer Science) through its structured, modern appearance, and professionalism. The layout is clean; all artifacts within the layout are organized and demonstrate an attention to detail regarding structure and development.

5. What does your homepage layout look like? 
The homepage uses a structured layout with the following distinct sections:
An upper navigation menu
A hero image with a description of my career, including my position and what I do
A place for my technical skills
A call-to-action button that links to my projects

6. What type of visual format will you use to present your project sections?
The project presentations will be presented through a grid-type format with card-style sections. Each section will have a title or name for the project along with a brief description about it. This way, all projects can be displayed in a logical order, while still providing users with an easy-to-scan structure.

7. Will this site be accessible via mobile devices? How will you ensure responsive design is effective?
Yes! This site is designed for access via mobile devices too. It uses several responsive design techniques such as flexible layouts and CSS Grid layouts, which let the site adjust properly to varying sizes of screens.

8. What will guide visitors to a visual priority among all sites?
Using headings, spacing, and section layouts directs visitors' attention to headings with greater weight, showing them where important sections are located, and using less weight to show supporting text.

9. What will provide cohesiveness to all pages of the star.com site?
The same navigation bar, color scheme, fonts, and layout style will keep all pages in perfect harmony.

10. How will you ensure that content on the star.com site is accessible for as many users as possible (high contrast, appropriate type size, and format)?
Content on the star.com site will be written with great attention by employing high contrast between text and background, appropriate type sizes that are readable for all types of users, and using traditional typeset structures to make content easily read.

11. Do you intend to include visual cues (icons, images, illustrations)? If so, what is your motivation for their use?
Simple icons and/or images may be used to represent individual projects and skills. The use of visual cues allows users to further engage with the website and improves the site's overall appeal.

12. Which portfolio websites influenced your design concept?
The Star.com website design concept was primarily inspired by modern developer portfolio sites, which feature minimalistic design, clear navigation, and an attractive display of projects.
